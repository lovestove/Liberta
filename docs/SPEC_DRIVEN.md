# MASTER SPECIFICATION: Liberta (The Neural Mesh VPN)
*Полный архитектурный и функциональный манифест проекта*

## 1. Архитектура Системы (High-Level Architecture)

Liberta представляет собой гибридное приложение, состоящее из низкоуровневого сетевого ядра и высокоуровневого пользовательского интерфейса, взаимодействующих через CGO-мост.

### 1.1. Core Layer (Go 1.22+ & sing-box)
Ядро отвечает за всю сетевую логику, шифрование, маршрутизацию и обфускацию.
- **Основа:** Форк проекта `sing-box`. Выбран за модульность и нативную поддержку современных протоколов (VLESS, Hysteria2, TUIC v5).
- **Интеграция:** Сборка осуществляется утилитой `gomobile bind`, которая генерирует `.aar` (Android Archive) с JNI-обертками.
- **Управление состоянием:** Ядро работает как стейт-машина, управляемая через gRPC-подобный бинарный протокол из Kotlin-кода для минимизации оверхеда сериализации JSON.
- **Tun Stack:** `gVisor` (написан на Go) для перехвата всего сетевого трафика устройства на уровне L3 (IP) и преобразования его в L4 (TCP/UDP) потоки для проксирования. Выбран вместо `lwIP` ради лучшей интеграции с Go-экосистемой.

### 1.2. Application Layer (Kotlin & Android SDK)
- **UI Framework:** Jetpack Compose.
- **Архитектура:** MVI (Model-View-Intent) или Clean Architecture + MVVM. MVI предпочтительнее для жесткого контроля состояний сложного UI (живой фон, переходы).
- **DI:** Dagger Hilt для предоставления экземпляров SingBoxController, DataStore и сетевых репозиториев.
- **Хранилище:** Jetpack DataStore (Preferences для настроек, Proto DataStore для кэширования списка серверов и их метрик).
- **Фоновая работа:** WorkManager для периодического обновления подписок (если доступно прямое соединение) и синхронизации P2P-узлов.

## 2. Алгоритмы Связности и Оптимизации (Zero-Downtime Engine)

Критическая задача проекта — мгновенный коннект и работа с огромными подписками (1500+ серверов).

### 2.1. Infrastructural Deduplication (Дедупликация 1500+ серверов)
Анализ «в лоб» 1500 серверов невозможен.
1.  **Парсинг подписки:** При загрузке подписки (`WHITE-CIDR-RU-all.txt` или `BLACK_VLESS_RUS.txt`) ядро парсит конфигурации.
2.  **IP-to-ASN Resolution:** Локальная встроенная база (MMDB) сопоставляет IP-адреса серверов с их ASN (Autonomous System Number, например, AS16276 - OVH).
3.  **Кластеризация:** 1500 серверов схлопываются в ~30-50 уникальных кластеров (подсетей/провайдеров).
4.  **Гейтвей-пробинг:** Ядро выбирает 1 случайный сервер из каждого кластера для теста. Если узел отваливается по таймауту (<800ms) или режет TLS-хендшейк, *весь* кластер помечается как "Dead".
5.  **Ранжирование:** Выжившие кластеры сортируются по исторической задержке (сохраняется в DataStore).

### 2.2. Parallel Connectivity Racing (Гонка Подключений)
Выбрав топ-5 кластеров, Liberta запускает гонку:
- Ядро инициирует 5 параллельных TLS/QUIC хендшейков.
- Вместо простого ping отправляется HTTP HEAD запрос к `generate_204` эндпоинту (например, `http://cp.cloudflare.com/generate_204`).
- Первый сервер, вернувший HTTP 204 No Content, объявляется победителем. Остальные горутины жестко отменяются (Context Cancellation).
- **Итог:** Поиск рабочего сервера занимает время, равное RTT самого быстрого узла + время хендшейка.

## 3. Революционные Технологии Обхода (Liberta Labs)

Эти функции активируются в специальном разделе настроек и реализуют беспрецедентный уровень защиты.

### 3.1. Phantom Call & WebRTC Steganography
Обход строгих «белых списков» (когда разрешены только локальные IT-гиганты, например ВК).
- **Инкапсуляция:** Трафик VPN оборачивается в датаграммы UDP.
- **Шейпинг:** Ядро модифицирует размеры пакетов и интервалы отправки (Inter-Packet Arrival Time), чтобы они в точности совпадали с профилем видеозвонка (например, кодек H.264 + Opus, битрейт ~1.5 Mbps).
- **Маскировка (SNI/STUN):** Хендшейк имитирует STUN/TURN запросы к серверам гигантов.
- **Serverless Relay:** В качестве точек входа (т.к. мы не можем арендовать серверы у ВК) используются Serverless-решения (Yandex Cloud Functions, Cloudflare Workers, GitHub Actions), чьи IP-адреса по умолчанию имеют высокий траст в РФ и часто попадают в белые списки.

### 3.2. Polymorphic DNA (Полиморфное Ядро)
Защита от активного пробинга и анализатора сигнатур (DPI).
- Вместо статических параметров шифрования (например, фиксированного `clientHello` в TLS), JIT-модуль ядра генерирует случайную, но криптографически валидную структуру пакетов.
- Длина padding-а (мусорных данных) динамически меняется для каждого пакета по псевдослучайному закону, ключ от которого согласовывается при хендшейке. DPI не может составить паттерн.

### 3.3. High-Density Social Steganography (Авто-обновление через картинки)
Решение проблемы «Как обновить список серверов, если заблокировано всё».
1.  **GitHub Actions Pipeline:** В репозитории настраивается CI/CD. Раз в час воркер скачивает свежие конфигурации.
2.  **Сжатие:** 1000 строк VLESS конфигов сжимаются алгоритмом Brotli (степень сжатия 9) до ~20 KB.
3.  **LSB Steganography:** Сжатый бинарный блоб шифруется AES-256-GCM. Затем с помощью алгоритма замены наименее значащего бита (Least Significant Bit) встраивается в пиксели изображения (PNG/Lossless WebP). Визуально картинка не меняется.
4.  **Распространение:** Воркер через API постит картинку в публичные паблики ВК или Telegram-каналы.
5.  **Приложение:** Liberta имеет захардкоженные ID пабликов. Если связи нет, она скачивает последнюю картинку, извлекает и расшифровывает LSB-данные, получая свежий конфиг.

### 3.4. Sovereign P2P Relay (Децентрализованный мост)
Создание mesh-сети из самих пользователей.
- **DHT Discovery:** Узлы находят друг друга через распределенную хеш-таблицу (Kademlia DHT, реализованную поверх UDP).
- **Роли:** Пользователь может нажать тумблер «Стать Суверенным Узлом» (Sovereign Node). Его устройство начинает слушать входящие соединения (если это позволяет NAT, используется STUN/TURN или Hole Punching).
- **Трафик:** Трафик соседа зашифрован ключом конечного сервера. Реле-узел *не может* расшифровать или прочитать трафик, он только пересылает зашифрованные пакеты (Onion Routing).
- **Homeostasis:** Реле-режим отключается автоматически, если заряд батареи падает ниже 30% или если пользователь переходит с Wi-Fi на лимитный мобильный интернет.

## 4. Дизайн, UI и UX (Liquid Glass Aesthetic)

Интерфейс должен транслировать «премиальность» и инновационность. Мы отказываемся от стандартных Material Design карточек.

### 4.1. Визуальный Язык
- **Стиль:** Liquid Glass (Жидкое стекло). Многослойный полупрозрачный интерфейс, сильное размытие заднего фона (Blur/Frosted Glass), мягкие тени (Drop Shadows с большим радиусом и низкой прозрачностью).
- **Типографика:** Геометрические гротески (Inter, Outfit) для заголовков, моноширинные шрифты для технических метрик (ping, IP).

### 4.2. Экраны и Компоненты
- **Main Screen:**
    - Топ-бар: Логотип (градиентный текст), кнопка Settings (переход в Labs), кнопка Share (вызов QR-кода).
    - Центр: Огромная кнопка активации.
    - Низ: Кастомный физичный тумблер (с пружинной анимацией) для переключения: `Белые списки` (обход блокировок локально) <-> `Черные списки` (полный VPN).
- **Анимация Подключения (Liquid Wave):**
    - В состоянии `Disconnected` интерфейс монохромный (оттенки серого и темно-синего).
    - При нажатии на `Connect` из-под кнопки вырывается анимированная волна (Canvas API + Path animation), которая окрашивает интерфейс в глубокие неоновые цвета (Electric Blue, Purple).
- **Adaptive Living UI (Живой Фон):**
    - Имплементация через кастомный `Modifier.drawWithCache` или `AndroidView` с OpenGL шейдером.
    - Фон представляет собой медленно движущиеся градиентные сферы. Скорость их движения и пульсация зависят от текущего битрейта сети. Больше трафика -> активнее фон.
    - **Eco Mode:** В настройках Labs можно переключить фон в статичный режим для экономии батареи.

## 5. Энергоэффективность и Стабильность (Homeostasis Engine)
- **WakeLocks:** Ядро минимизирует использование CPU WakeLocks. Сетевые пробы (ping) синхронизируются с окнами активности Android (Doze Mode).
- **Smart Warm-up:** При запуске приложения (Activity `onCreate`), ядро *сразу* инициирует проверку топ-3 серверов. К моменту, когда пользователь нажмет "Connect", маршрут уже известен. Подключение происходит за 0.1 секунды.
- **Redundant Shield (Опционально):** Режим отправки дублирующих UDP-пакетов через 2 разных сервера для критичных сессий (VoIP звонки). Гарантирует 0% Packet Loss даже при падении одного из серверов.
